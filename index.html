    Sua Porta de Entrada

    O Claude Code RODA no terminal. Se você não sabe abrir e usar o terminal, não conseguirá usar nada do que vem depois.

    O QUE É O TERMINAL?

    Definição Simples

    O terminal é a interface de texto do seu computador. Em vez de clicar em ícones, você digita comandos e o computador executa.

    Pense no terminal como enviar mensagens de texto para o seu computador. Você escreve o que quer, ele faz.

    Onde Encontrar

    Mac: Abra o app Terminal (busque no Spotlight com Cmd+Espaço) ou instale o iTerm2 (mais bonito)

    Windows: Abra o PowerShell ou Windows Terminal (baixe na Microsoft Store)

    VS Code: Pressione Ctrl+` (crase) para abrir o terminal integrado

    # Exemplo: o que aparece quando você abre o terminal macbook@usuario ~ % ← este é o "prompt", onde você digita macbook@usuario ~ % ls Desktop Documents Downloads Pictures # ↑ Você digitou "ls" e ele listou suas pastas
    DICA: O terminal parece assustador no começo, mas você só precisa de ~10 comandos para 90% das tarefas!

    MÓDULO 02

    GIT & GITHUB

    Controle de Versão — Seu CTRL+Z Infinito

    Imagine ter um histórico completo de TUDO que já mudou no seu projeto, podendo voltar a qualquer ponto.

    Claude Code usa Git CONSTANTEMENTE. Cada commit, cada branch, cada PR passa pelo Git. Entender Git é entender metade do Claude Code.

    O QUE É GIT?

    Definição

    Git é um sistema de controle de versão — ele salva "fotos" do seu projeto ao longo do tempo, permitindo que você volte a qualquer versão anterior.

    Analogia: Google Docs

    Sabe quando o Google Docs mostra "Histórico de versões" e você pode ver todas as alterações que já fez? Git faz a mesma coisa, mas para todos os arquivos do projeto inteiro, de forma muito mais poderosa.

    Arquivo v1
    Versão inicial
    →
    Commit #1
    "cria homepage"
    →
    Arquivo v2
    Com mudanças
    →
    Commit #2
    "adiciona login"
    →
    Arquivo v3
    Mais mudanças

    ↑ Cada "commit" é um ponto de salvamento. Você pode voltar a qualquer um deles.

    ⚠️ Sem Git, se você estragar algo, perdeu. Com Git, você volta ao commit anterior e tá tudo bem.

    GIT vs GITHUB — QUAL A DIFERENÇA?

    GIT

    Ferramenta local instalada no seu computador

    É o motor — faz o trabalho de salvar versões, criar branches, fazer merge

    Funciona sem internet

    Analogia: É o motor do carro

    GITHUB

    Site na nuvem (github.com) para guardar e compartilhar projetos

    É a garagem online — guarda seu projeto na internet, permite colaboração

    Precisa de internet

    Analogia: É a garagem onde você guarda o carro

    Resumo Visual

    Seu PC
    (Git local)
    ← push / pull →
    GitHub
    (nuvem)
    ← clone →
    PC do Colega
    (Git local)

    Alternativas ao GitHub: GitLab, Bitbucket — mesma ideia, sites diferentes. GitHub é o mais popular.

    BRANCHES — O CONCEITO MAIS IMPORTANTE DO GIT

    Branches permitem trabalhar em coisas diferentes ao mesmo tempo sem bagunçar o projeto principal.

    Analogia: Documento com Rascunhos

    Imagine que você tem um TCC pronto (branch main). Você quer testar uma nova seção, mas não quer arriscar estragar o que já está bom.

    Você cria uma cópia (branch nova-secao), trabalha nela à vontade. Se ficou bom, junta (merge) com o original. Se ficou ruim, deleta a cópia e o original continua intacto.

    # Visualizando branches como linhas do tempo: main: ●───●───●───●───●───● (versão estável, produção) \ ↗ feature/login: ●───●───●───● (trabalho isolado, depois junta) \ fix/bug-123: ●───● (correção de bug, depois junta) # Cada ● é um commit # As linhas mostram onde cada branch divergiu e (opcionalmente) voltou

    Branch main

    A branch principal. É o que está "no ar", funcionando. Nunca trabalhe direto nela!

    Feature Branches

    Branches temporárias para cada tarefa. Ex: feature/login, fix/bug-404. Quando termina, faz merge.

    O Claude Code cria e gerencia branches automaticamente quando você pede. Mas entender o conceito faz você dar instruções melhores.

    MERGE E CONFLITOS — O BICHO-PAPÃO DO GIT

    O Que é um Conflito?

    Acontece quando duas pessoas (ou branches) modificam a mesma linha do mesmo arquivo. O Git não sabe qual versão manter e pede sua ajuda.

    # Quando há conflito, o Git marca assim no arquivo: <<<<<<< HEAD const titulo = "Meu Site Incrível" ======= const titulo = "Site Fantástico" >>>>>>> feature/novo-titulo # HEAD = sua versão atual # feature/novo-titulo = a versão que está tentando juntar # Você escolhe qual manter (ou combina as duas), remove os marcadores e commita

    Como Resolver

    1. Abra o arquivo conflitante

    2. Escolha qual versão manter (ou combine)

    3. Remova os marcadores <<<<<<<, =======, >>>>>>>

    4. git add arquivo → git commit

    🤖 Claude Code resolve conflitos automaticamente! Basta dizer: "Resolva os conflitos de merge" e ele analisa, escolhe a melhor resolução e commita.

    PULL REQUESTS — O CORAÇÃO DA COLABORAÇÃO

    O Que é um Pull Request (PR)?

    É um pedido para juntar sua branch na branch principal. Antes de juntar, outras pessoas podem revisar seu código, comentar e sugerir melhorias.

    1. Cria branch
    →
    2. Faz commits
    →
    3. Push p/ GitHub
    →
    4. Abre PR
    →
    5. Review
    →
    6. Merge!

    O PR inclui:

    • Título e descrição das mudanças

    • Lista de commits incluídos

    • Diff (todas as linhas alteradas)

    • Espaço para comentários

    • Status de testes automáticos

    Por que PRs são importantes?

    • Revisão de código previne bugs

    • Documentação automática das mudanças

    • Testes rodam antes de juntar

    • Histórico de decisões técnicas

    • É assim que equipes profissionais trabalham

    O Claude Code cria PRs completos com título, descrição e test plan usando o comando /pr-comments e o GitHub CLI (gh).

    MÓDULO 10

    SUPERPOWERS

    Skills Avançadas para Claude Code

    Transforme o Claude Code em um especialista disciplinado.

    O QUE SÃO SKILLS?

    Skills são "roteiros estruturados" que o Claude Code segue para executar tarefas de forma disciplinada. Em vez de improvisar, ele segue um processo testado.

    ❌ Sem Skills

    "Corrija o bug"

    → Claude improvisa, pode ir por caminhos errados, não verifica sistematicamente

    ✅ Com Skills

    "Corrija o bug"

    → Skill de debugging ativa → investiga sistematicamente → isola causa → corrige → verifica → confirma

    Analogia

    Pense em Skills como receitas de cozinha: passos definidos para um resultado consistente. Um chef pode improvisar, mas seguir a receita garante que o prato saia bom toda vez.

    Instalando o Superpowers

    # Dentro do Claude Code, execute: /install-plugin superpowers@superpowers-marketplace # Após instalar, as skills ficam disponíveis como slash commands # Prefixo: /superpowers:
    Skills se conectam entre si: brainstorming → writing-plans → executing-plans → verification → code-review. É um fluxo completo de desenvolvimento profissional.
